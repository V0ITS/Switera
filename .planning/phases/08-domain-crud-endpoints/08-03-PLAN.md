<objective>
Port the Permintaan (request) domain from store.js to a real REST API (API-01) with Zod validation (API-02) and RBAC (AUTH-03). store.js exposes getPermintaan, hasPermintaanDuplikat, addPermintaan, updatePermintaan, removePermintaan. The tricky part is the snake_case <-> camelCase mapping: the frontend (Phase 9) calls these with store.js's snake_case shapes (kota, tanggal_permintaan, tanggal_input, jumlah_permintaan, keterangan), but Prisma uses camelCase (kotaNama, tanggalPermintaan, tanggalInput, jumlahPermintaan, keterangan). The API contract preserves the store.js snake_case shape (store-as-seam, FE-01) and the service maps to/from Prisma.

NOTE: addPermintaan in store.js ALSO pushes a notification + an anomaly notification + an activity-log entry. Those server-side side effects are LOGIC-03 and are implemented in 08-05, which extends this service's mutations. This plan delivers the pure CRUD + duplicate-check; 08-05 layers the activity/notification generation on top.

Purpose: Permintaan is the highest-volume CRUD domain and the source data for the ranking engine (08-04). Getting its shape contract exactly right keeps Phase 9's store-as-seam swap mechanical.
Output: permintaanService.js, permintaanSchemas.js, permintaanRoutes.js, verify script, router mounted.
</objective>
