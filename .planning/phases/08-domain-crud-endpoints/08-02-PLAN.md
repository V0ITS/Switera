RBAC contract (from Phase 7 SUMMARY 07-02, the established copy-paste pattern):
```
router.METHOD(path, requireAuth, requireRole("Admin"), handler)
```
requireAuth attaches req.user = { id, username, role }; requireRole(...roles) 403s if req.user.role is outside the list.

Validation + error handling (from 08-01):
- `import { validate } from "../middleware/validate.js";` -> `validate(schema)` 400s with field-level errors and allowlists req.body
- A central errorHandler is mounted LAST in index.js. Route handlers should be `async` and forward thrown service errors via `next(err)` (or wrap in try/catch and call next(err)). Do NOT format service errors inline — let errorHandler map them.

kotaService.js (Phase 6 — EXISTING, do not rewrite the working functions):
- `getDaftarKota()` -> Kota[]
- `getKotaReferenceCounts(nama)` -> { permintaanCount, keputusanCount }
- `updateKota(namaLama, { nama, kapasitas })` -> cascade-rename, throws "Kota dengan nama tersebut sudah ada." on collision
- `hapusKota(nama)` -> block-delete, throws "Kota ... tidak bisa dihapus karena masih digunakan ..." when referenced
- MISSING (this plan adds it): `tambahKota({ nama, kapasitas })`

src/store.js parity reference (the snake_case/shape the frontend expects in Phase 9):
- store.tambahKota({ nama, kapasitas }) throws "Kota dengan nama tersebut sudah ada." on duplicate
- store.getKapasitasKota(nama) -> number | null (derivable from GET /kota; no separate endpoint needed)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/curl -> /kota, /stok-tbs | Untrusted body + JWT; RBAC and Zod govern entry |
| route -> kotaService -> Prisma | Validated DTO only; cascade-rename runs in a Phase 6 transaction |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-K-RBAC | Elevation of Privilege | POST/PUT/DELETE /kota, PUT /stok-tbs | mitigate | requireRole("Admin") on every write; reads require only requireAuth; verified by 403 assertions in kotaRoutes.verify.mjs |
| T-08-K-MASS | Tampering | POST/PUT /kota body | mitigate | validate(kotaCreateSchema/kotaUpdateSchema) allowlists nama+kapasitas only; handler passes the parsed DTO, never raw req.body, into kotaService |
| T-08-K-INJECT | Injection | city nama (free text) -> Prisma where/data | accept | Prisma parameterizes; no raw SQL; city name flows only into typed Prisma calls — verified non-issue |
| T-08-K-RENAME | Tampering (integrity) | PUT /kota cascade-rename | mitigate | Reuses Phase 6 updateKota's `$transaction` (all-or-nothing); duplicate-name guard throws -> 409 via errorHandler |
</threat_model>
