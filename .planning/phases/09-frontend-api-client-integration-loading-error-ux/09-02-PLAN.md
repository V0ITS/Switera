<objective>
Convert the kota (city) and stok (TBS stock) domain of `store.js` from in-memory/localStorage to the Phase 8 REST API, and update its sole consuming page `ManajemenKota.jsx` to await the now-async mutators. This is the first per-domain application of the binding pattern decided in 09-01.

Purpose: Migrate the Admin city/stock management slice (FE-01) with full loading/error UX (FE-02) and zero visual change (FE-03).
Output: Reworked kota/stok internals of `src/store.js` and updated `ManajemenKota.jsx` handlers.

**Pattern reference (do NOT re-decide):** This plan reuses the hydrated-cache + Promise-returning-mutator + `runMutation` loading/error mechanism and the apiClient established in 09-01. See 09-01-PLAN.md objective ("THE CENTRAL ARCHITECTURAL DECISION", "LOADING / ERROR UX DECISION"). Reads stay synchronous against the cache; mutators `await apiFetch`, write the server response into the cache, then `notify()`.

Endpoints used (verified against server/src/routes/kotaRoutes.js, stokRoutes.js):
- GET /kota → `[{ nama, kapasitas }]` (auth)
- GET /kota/:nama/references → `{ permintaanCount, keputusanCount }` (auth)
- POST /kota body `{ nama, kapasitas }` → full updated list (201, Admin)
- PUT /kota/:nama body `{ nama, kapasitas }` → full updated list (Admin)
- DELETE /kota/:nama → full updated list (Admin); 409 with Indonesian message when referenced
- GET /stok-tbs → `{ stokTbs }` (auth)
- PUT /stok-tbs body `{ stokTbs }` → `{ stokTbs }` (Admin)
</objective>
