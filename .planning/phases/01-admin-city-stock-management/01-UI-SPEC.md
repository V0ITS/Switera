## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | `var(--text-sm)` = 13px | `var(--font-weight-normal)` = 400 | `var(--leading-normal)` = 1.5 |
| Label (field labels, table headers) | `var(--text-xs)` = 12px, uppercase, `var(--tracking-wider)` | `var(--font-weight-semibold)` = 600 | 1.2 (tight, single line) |
| Heading (page title `h1`, `Modal` title `h2`) | `var(--text-2xl)` = 26px (page) / `var(--text-lg)` = 18px (modal) | `var(--font-weight-bold)` = 700 (page) / `var(--font-weight-semibold)` = 600 (modal) | `var(--leading-tight)` = 1.2 |
| Display (TBS stock summary number) | `var(--text-2xl)` = 26px, `var(--font-mono)` | `var(--font-weight-bold)` = 700 | `var(--leading-tight)` = 1.2 |

These map directly to `src/tokens.css:42-60`. Do not introduce font sizes outside this set (`--text-2xs` through `--text-4xl` already cover every case used elsewhere in the app; this phase only needs `--text-xs`, `--text-sm`, `--text-lg`, `--text-2xl`).

Numeric values (TBS stock tonnage, city capacity in the table) use `var(--font-mono)` with `fontVariantNumeric: "tabular-nums"` — matches `Tabel.jsx:263-264` convention for the `numeric: true` column flag.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--color-bg)` #0d0d0d / `var(--color-surface)` #161616 | Page background, `Card` and `Tabel` surfaces |
| Secondary (30%) | `var(--color-surface-2)` #1c1c1c / `var(--color-surface-3)` #242424 | Table header row, input fields, `Modal` body background |
| Accent (10%) | `var(--color-primary)` #2d6a4f | Reserved for: primary CTA ("+ Tambah Kota" button), save/submit buttons in modals, active sort-column indicator in `Tabel`, focus ring (`--color-primary-glow`) on inputs |
| Destructive | `var(--color-danger)` #e5484d | Reserved for: "Hapus" row action button (`aksi-btn-delete`), "Ya, Hapus" confirm button (`tombol-bahaya`), delete-blocked notice border/icon, inline duplicate-name error text |

Accent reserved for: the single primary CTA button per view ("+ Tambah Kota", "Simpan Perubahan" inside Add/Edit modal, "Simpan" inside the TBS stock edit modal), the active/hovered table sort arrow, and input focus rings. Accent is never used for secondary actions, table chrome, or static labels.

Second semantic color (destructive only): `var(--color-danger)` is used exclusively for the delete row action, delete confirmation button, and the delete-blocked modal (D-01) — never for any non-destructive state. Duplicate-name validation errors (D-05/D-06) also use `var(--color-danger)` for inline text, consistent with `ManajemenData.jsx`'s `errorStyle` pattern, because a validation error is the same semantic class as "stop, this is wrong" even though it's not a destructive action — this matches existing precedent (`InputData.jsx`, `ManajemenData.jsx` both use `--color-danger` for inline field errors).

This is a single-theme-pair system: dark (default) and light (`:root[data-theme="light"]`) variants are both already defined in `tokens.css` — this phase does not add new color tokens, only consumes the existing ones. No new hex values anywhere in this phase's UI.

## Interaction & Layout Notes (brownfield-specific, supplements template)

These are not in the standard template but are necessary contract details given this phase builds directly on existing precedent:

**Page layout (single page, two zones, per D-03/D-04):**
1. `PageHeader` — title "Manajemen Kota", description chip, `aksi` slot holds "+ Tambah Kota" button (top-right, matches `ManajemenData.jsx` header pattern where `aksi` holds the search input instead).
2. TBS stock summary `Card` (or `MetricCard size="lg"`) — positioned above the city table, full width or alongside it in a 2-column grid on wide viewports (`>900px`), stacking to 1 column below that (reuse `.bento-grid`/`.app-grid-2` responsive helpers already in `tokens.css:277-307` if a 2-column layout is chosen; otherwise single full-width stacked `Card` is equally acceptable and simpler — Claude's Discretion per CONTEXT.md, default to single-column stacked unless the executor finds a clean 2-column fit).
3. City table `Card` containing `SectionHeader` ("Daftar Kota — Menampilkan N kota") + `Tabel` or `EmptyState`.

**Table columns (city list):**
| Key | Label | Notes |
|-----|-------|-------|
| `nama` | Nama Kota | text, sortable |
| `kapasitas` | Kapasitas (ton) | `numeric: true`, mono font, sortable |
| (action column via `aksi` prop) | Aksi | Edit (`aksi-btn-edit`) + Hapus (`aksi-btn-delete`), same as `AksiTabelButtons` pattern in `ManajemenData.jsx:23-57` |

**New icon required:** `Layout.jsx`'s `IkonMenu` switch (lines 114-204) has no existing case for a "city" concept. Add a new case `"city"` (simple two-stroke building/pin glyph, `strokeWidth="1.5"`, same `iconStyle` sizing as all other cases) and reference it as `{ key: "manajemen-kota", label: "Manajemen Kota", icon: "city" }` in `menuByRole.Admin` (`src/utils/navigation.js:8-13`). Do not reuse the `"database"` icon (already used by "Manajemen Data" — would be visually ambiguous between the two Admin menu items).

**Form validation rendering:** Follow the `validate(form) -> {field: message}` pattern exactly as in `InputData.jsx`/`ManajemenData.jsx` — errors computed on every `handleChange` and re-checked before submit, rendered as a `<p style={errorStyle}>` directly under the offending field (never a toast for field-level validation; toasts are reserved for post-mutation success/failure feedback per D-05).

**TBS stock edit interaction (D-04):** Card-with-edit-button, not inline-editable, not a table row. Clicking "Edit" opens a `Modal` with a single numeric field ("Stok TBS Tersedia (ton)"), pre-filled with the current value, Cancel/Simpan buttons. This deliberately differs from `ManajemenData.jsx`'s inline-click-to-edit pattern for `jumlah_permintaan` — D-04 explicitly rejected inline quick-edit for this value.
