<objective>
Implement automatic cross-client data synchronization via interval polling (SYNC-01), built directly on top of the Phase 9 hydrated-cache architecture.

The mechanism is deliberately minimal: a `setInterval` that re-runs the existing `store.hydrate()` Promise.all over every domain loader on a fixed cadence. Each tick rewrites the in-memory cache with the server's authoritative state and fires the existing `notify()`, so every subscribed page re-renders with fresh data — no manual refresh, no new per-page wiring, no WebSocket/SSE (explicitly out of scope per REQUIREMENTS.md SYNC-02 deferral).

Purpose: Two or more users logged in simultaneously across any combination of the 3 roles converge on the same data within seconds of any change, while the authorization/concurrency guarantees from Phases 7/8 remain untouched (this plan adds NO new route and relaxes NO RBAC/validation check — it only calls existing loaders).

Output: `startPolling()`/`stopPolling()`/`pollTick()` in `src/store.js`; polling lifecycle wired into `src/App.jsx`'s existing `userAktif?.username` effect alongside the current `store.hydrate()` call.
</objective>
