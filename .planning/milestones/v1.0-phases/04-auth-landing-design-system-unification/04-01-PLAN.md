## Phase Goal

**As a** Switera user opening the login or register screen, **I want to** see auth forms that use the same buttons and visual language as the rest of the app, **so that** the product feels consistent and trustworthy from the very first screen.

## Artifacts this phase produces

This plan creates no new symbols, files, functions, or exports. It is a refactor-in-place that:
- Adds `import Tombol from "../components/Tombol";` to `src/pages/Login.jsx` and `src/pages/Register.jsx`
- Replaces the `.auth-submit-btn` `<button>` element in each file with a `<Tombol type="submit" ... />` call
- Removes the now-unused `submitRipple` `useRipple()` hook instance and its `RippleSpans` usage from each file (the manual submit-button ripple), since `Tombol` provides its own ripple internally
