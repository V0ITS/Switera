## Content Visibility & Data Contract

This is the core contract for Phase 2. Laporan page now conditionally renders role-appropriate data instead of always showing decision-focused content.

### For Manajer Distribusi (`snapshot.roleAktif === "Manajer Distribusi"`)

**Heading:**
- Page title: "Laporan Distribusi" (unchanged)
- Description: "Riwayat keputusan bersifat permanen dan tetap tersedia untuk audit periode sebelumnya." (unchanged)

**Content Structure:**
1. **Filters & Actions** (unchanged):
   - Periode pills: "Minggu ini" / "Bulan ini"
   - Ekspor CSV button (exports riwayatKeputusan filtered by periode)
   - Both enabled/visible

2. **Primary Section — Riwayat Keputusan Table**:
   - **Visibility:** Show when `filteredRiwayat.length > 0`
   - **Card Header:** "Riwayat Keputusan"
   - **Description text:** "Seluruh keputusan yang pernah dibuat, termasuk yang dibatalkan, tetap ditampilkan pada laporan."
   - **Columns:**
     | Column | Source Field | Format | Numeric |
     |--------|--------------|--------|---------|
     | Tanggal | tanggal_keputusan | formatDate() | false |
     | Kota Tujuan | kota_tujuan | literal | false |
     | Volume TBS | volume_tbs | formatTonase() | true |
     | Diputuskan Oleh | diputuskan_oleh | literal | false |
     | Status | status | Badge component | false |
   - **Empty state:** "Belum ada riwayat keputusan pada periode yang dipilih."

3. **Secondary Section — Tren Permintaan Chart**:
   - **Visibility:** Show when `chartConfig.labels.length > 0 && datasets.length > 0`
   - **Card Header:** "Tren Permintaan per Kota"
   - **Chart type:** Line chart (Chart.js)
   - **Data source:** `filteredPermintaan` (not keputusan), grouped by city and date
   - **Chart config:** Per `chartConfig` computed from filteredPermintaan
   - **Empty state:** "Belum ada data tren permintaan pada periode yang dipilih."

4. **No-Data State** (when all data empty):
   - Message: "Tidak ada data pada periode yang dipilih."

### For Tim Logistik (`snapshot.roleAktif === "Tim Logistik"`)

**Heading:**
- Page title: "Laporan Status Distribusi" (differs from Manajer)
- Description: "Pantau progres distribusi aktif dan riwayat pengiriman per periode." (differs from Manajer)

**Content Structure:**
1. **Filters & Actions**:
   - Periode pills: "Minggu ini" / "Bulan ini" (unchanged)
   - Ekspor CSV button: Still enabled, but exports status/delivery-focused data (see below)
   - Both enabled/visible

2. **Primary Section — Status Distribusi Table** (replaces Riwayat Keputusan):
   - **Data source:** `keputusan` (not riwayatKeputusan), filtered by `periode` range
   - **Visibility:** Show when `filteredKeputusan.length > 0`
   - **Card Header:** "Distribusi Aktif"
   - **Description text:** "Seluruh distribusi dengan detail armada dan status pengiriman untuk audit periode sebelumnya."
   - **Columns:**
     | Column | Source Field | Format | Numeric |
     |--------|--------------|--------|---------|
     | Tanggal | tanggal_keputusan | formatDate() | false |
     | Kota Tujuan | kota_tujuan | literal | false |
     | Volume TBS | volume_tbs | formatTonase() | true |
     | Armada | armada + eta | "{armada} · ETA {formatDate(eta)}" or "-" | false |
     | Status | status | Badge component | false |
   - **Empty state:** "Belum ada distribusi aktif pada periode yang dipilih."

3. **Secondary Section — Status Distribution Chart** (replaces Tren Permintaan):
   - **Chart type:** Bar or Doughnut chart (new, executor decides based on clarity)
   - **Data source:** Count of keputusan by status (menunggu, dalam-pengiriman, selesai)
   - **Labels:** Status label map from StatusDistribusi.jsx
   - **Visual:** Shows proportion of distribution across status states
   - **Empty state:** "Belum ada data status distribusi pada periode yang dipilih."

### Fallback Behavior

If `snapshot.roleAktif` is neither "Manajer Distribusi" nor "Tim Logistik" (edge case, should not happen):
- Default to **Manajer Distribusi** view (current behavior)
- Log no warning — assume role consistency handled upstream

## Interaction Contract

All interactions remain unchanged from Phase 1 Laporan. New role-conditional behavior is **data-only**, not interaction-only.

| Action | Behavior | Change? |
|--------|----------|---------|
| Switch periode (pills) | Re-filter all visible tables/charts by date range | No change |
| Click Ekspor CSV | Download filtered data to CSV | Yes: Manajer Distribusi exports keputusan; Tim Logistik exports keputusan + armada |
| Render Laporan (mount) | Subscribe to store, render current roleAktif view | Yes: conditional on roleAktif; layout unchanged |
| Role change (user logout/login) | Store notifies listeners, Laporan re-renders entire page with new roleAktif | Yes: conditional rendering; no navigation |

Both roles use the **same page structure**, different data:

```
┌─ PageHeader ───────────────────────┐
│ Title + Description (role-specific)│
│ Period pills | Ekspor CSV button    │
└────────────────────────────────────┘
        ↓
   ┌─ No-Data Empty State (if no data)
   │
   └─ Data Sections:
      ┌─ Card: Riwayat Keputusan / Distribusi Aktif
      │ (role-specific table)
      │ Column 1: Tanggal
      │ Column 2: Kota Tujuan
      │ Column 3: Volume TBS
      │ Column 4: Diputuskan Oleh / Armada + ETA
      │ Column 5: Status
      └─
      ┌─ Card: Tren Permintaan / Status Pengiriman
      │ (role-specific chart)
      │ Line chart / Bar or Doughnut chart
      └─
```

**Responsive behavior:** Existing Card and Tabel components handle responsive layout; no changes required.
