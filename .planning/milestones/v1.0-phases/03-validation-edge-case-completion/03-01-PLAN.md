<objective>
Close VALID-01 (Login field-level inline errors) and VALID-04 (Register ID convention) — the two auth-page validation/convention gaps from Phase 3.

Purpose: Login currently shows one generic error string for every failure mode (blank fields, wrong username, wrong password, missing role), forcing the user to guess which field is wrong. Register currently mints account IDs with `Date.now()`, diverging from the `getNextId(items, prefix)` convention used everywhere else in `store.js` (notifications, activity log, permintaan, keputusan).
Output: `Login.jsx` renders per-field inline errors (`errors.username`, `errors.password`, `errors.role`) using the existing `ErrorText` component from `AuthShared.jsx`. `store.js` exposes a new `getNextAkunId()` method. `Register.jsx` calls it instead of `Date.now()`.
</objective>
