<objective>
Close VALID-02 (StatusDistribusi armada/ETA required validation) and VALID-03 (InputData empty-city-list explanatory message) — the two data-entry-page validation/convention gaps from Phase 3.

Purpose: StatusDistribusi's update-status modal currently lets armada and ETA stay blank even when status is set to "dalam-pengiriman", silently persisting incomplete delivery data. InputData currently renders an empty, unexplained `<select>` dropdown when no cities exist yet (e.g. on a fresh install before Admin has added any city via Phase 1's Manajemen Kota), giving the user no indication why the dropdown has nothing in it.
Output: `StatusDistribusi.jsx` gains a `modalErrors` state + `validateModalForm()` function that blocks `saveStatus()` and renders inline errors when armada/ETA are blank under "dalam-pengiriman". `InputData.jsx` renders an explanatory message and skips the select dropdown entirely when `daftarKota.length === 0`, and blocks form submission in that state.
</objective>
