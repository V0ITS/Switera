# Quick Task 260704-2y2: Ringkasan AI di halaman Laporan (AI-1)

## Objective

Pengguna di halaman Laporan dapat menekan tombol "Buat Ringkasan AI" dan menerima ringkasan naratif Bahasa Indonesia atas data laporan periode terpilih, dihasilkan oleh Claude API di sisi server (kunci API tidak pernah menyentuh browser).

## Design decisions

- **SDK resmi** `@anthropic-ai/sdk` di `server/` (bukan raw fetch) — mengikuti panduan resmi Claude API; model `claude-opus-4-8`, adaptive thinking, non-streaming (output ringkasan pendek).
- **Endpoint** `POST /laporan/ringkasan` body `{ periode: "semua" | "minggu-ini" | "bulan-ini" }`, `requireAuth` tanpa `requireRole` (semua role membaca laporan, konsisten dengan `/kpi`).
- **Sudut pandang per role diturunkan dari `req.user.role`** (Tim Logistik → fokus status pengiriman; selainnya → fokus riwayat keputusan/manajerial) — bukan dari body, agar tidak bisa dipalsukan.
- **Filter periode di server** — port `getPeriodRange`/`isDateInRange` dari `src/utils/distribusi.js` (logika sama persis) agar angka ringkasan cocok dengan yang dilihat pengguna.
- **Kunci API** `ANTHROPIC_API_KEY` di `server/.env` (gitignored). Jika belum diset → error 503 berbahasa Indonesia yang jelas, bukan crash.
- **Store method stateless** `buatRingkasanLaporan(periode)` — hasil dipegang state lokal komponen (ringkasan bukan data domain yang perlu polling/cache).

## Tasks

### Task 1 — Backend: service + route ringkasan
**Files:** `server/package.json`, `server/src/services/ringkasanService.js`, `server/src/schemas/ringkasanSchemas.js`, `server/src/routes/ringkasanRoutes.js`, `server/src/index.js`
**Action:** Install `@anthropic-ai/sdk`; buat service yang mengumpulkan permintaan/keputusan/riwayat/kota/stok + KPI via service layer yang ada (tanpa akses Prisma langsung), memfilter per periode, menyusun prompt ringkas Bahasa Indonesia, memanggil Claude, dan mengembalikan `{ ringkasan, periode, dibuatPada }`; schema Zod untuk `periode`; route `POST /laporan/ringkasan` dengan `requireAuth + validate`; mount di `index.js` di atas `errorHandler`.
**Verify:** `node --check` semua file baru; server boot; panggilan tanpa `ANTHROPIC_API_KEY` menghasilkan 503 dengan pesan Indonesia (bukan 500 generik).
**Done:** Endpoint terdaftar, error path bersih, tidak ada kunci API di frontend.

### Task 2 — Frontend: section "Ringkasan AI" di Laporan.jsx
**Files:** `src/store.js`, `src/pages/Laporan.jsx`
**Action:** Tambah `buatRingkasanLaporan(periode)` di store (async, via `apiFetch`); di Laporan.jsx tambah Card "Ringkasan AI" (komponen shared: Card, SectionHeader, Tombol) dengan tombol buat ringkasan, state loading ("Membuat ringkasan…", tombol disabled), state error (teks `--color-danger`), dan hasil naratif; ringkasan/error di-reset saat periode berganti agar tidak menyesatkan.
**Verify:** Vite build/HMR tanpa error; alur klik → loading → hasil/error tampil.
**Done:** Section tampil untuk kedua role, konsisten dengan design system (token + komponen shared, tanpa styling baru ad-hoc).

## must_haves

- Endpoint `POST /laporan/ringkasan` requireAuth, validasi Zod, error 503 Indonesia saat kunci belum diset
- Panggilan Claude hanya dari server; `ANTHROPIC_API_KEY` hanya di `server/.env`
- Section Ringkasan AI di Laporan.jsx dengan loading/error state memakai komponen shared
